# Memory Files

> 一句话定义：[[Anthropic]] 给 [[Claude]] 造的"内置个人 Wiki"——按话题/项目自动组织成结构化文档，按需选择性读取，是 [[LLM Wiki]] 理念的**消费级产品化**。

## 基本信息

- **类型**：产品功能 / 记忆系统
- **厂商**：[[Anthropic]]
- **状态**：灰度测试中（截至 2026-05-26）
- **取代对象**：Claude "经典记忆"模式（滚动便签纸）
- **关联实体**：[[Dreams]]（自动维护机制）· [[Conway]]（运行时载体）· [[CLAUDE.md]]（关系待定）
- **关联概念**：[[LLM Wiki]] · [[RAG]] · [[Maintain 维护]] · [[REM 睡眠式记忆整合]]

## 关键事实

- 信源：TestingCatalog 爆料 + Anthropic 内部披露 (来源: [[20260525-知乎-Claude永久大脑真的来了]])
- 正式上线后，用户可在「经典记忆」与「文件记忆」之间自由切换
- 设计哲学："按需精确投喂"取代"全量记忆暴力灌入"
- 记忆文件可像 Wiki 词条一样浏览、修改、删除——**不想被记住直接删文件**
- 基于文件系统的记忆并非首创（开源圈有 Hermes 等），但 Anthropic 是首个做成**消费级原生功能**的玩家

## 工作机制（已披露细节）

1. Claude 在对话中根据话题/项目/上下文，**自动**编写并组织成结构化文档
2. 未来对话涉及相关主题时，**选择性读取**对应文件，而非全量塞入上下文
3. 文件可被用户直接编辑

## 三大颠覆性优势

| 优势 | 说明 |
|---|---|
| 无限扩展 | 文件系统记忆理论上不受单一摘要长度瓶颈 |
| 按需检索 | 聊代码只调技术文档，聊旅行只调出行偏好 |
| 可读可改 | 用户像编辑 Wiki 一样掌控自己的记忆 |

## 与本知识库的强对照

| 维度 | [[LLM Wiki]]（本知识库） | Memory Files |
|---|---|---|
| 载体 | [[Obsidian]] 本地 Markdown | Anthropic 黑盒 |
| 编写者 | [[Hana]] / 其他 Agent | Claude 自动 |
| 触发 | 用户说"知识图谱..." | Claude 主动判断 |
| 用户可控 | 文件完全可见可改 | 文件可见可改 |
| 跨 Agent | 中立（任何 Agent 都能用） | 仅 Claude |
| 业务定制 | AGENT.md 可自定义协议 | 黑盒规则 |
| 数据所有权 | 完全本地 | 云端 |
| Git 可控 | 是 | 否 |

**结论**：Memory Files 的出现**不取代**本知识库——本知识库的 Obsidian 本地化、协议可定制、跨 Agent 中立是关键护城河。
