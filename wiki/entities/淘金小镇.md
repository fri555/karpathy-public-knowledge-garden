# 淘金小镇

> 一句话定义:[[卡尔的AI沃茨]] 做的 Skill 榜单情报 Skill ——每天爬 [[ClawHub]] 下载榜 Top100,**把易变的公开信息源,变成可被 Agent 复盘的时间线情报网**。

## 基本信息

- **名称**:淘金小镇(skillrush-town)
- **类型**:Skill / 开源项目 / 自动化情报工具
- **仓库**:https://github.com/LearnPrompt/skillrush-town
- **作者**:[[卡尔的AI沃茨]](LearnPrompt 团队)
- **数据源**:[[ClawHub]] 下载榜 Top100
- **技术栈**:Codex(`/goal` 模式)+ [[Convex]] API
- **输出**:每日榜单快照 + 对比报告 + GitHub Pages 静态网页
- **关联概念**:[[Skill 榜单情报化]] · [[Goal-Driven Execution]] · [[复刻用户视角拿数据]] · [[知识复利]]

## 关键事实

- 抓取频率:每日一次 (来源: [[20260527-知乎-卡尔-淘金小镇Skill榜单情报项目]])
- 数据获取方式:复刻 ClawHub 网页自己用的 [[Convex]] API `skills:listPublicPageV4`,通过 `nextCursor` 分页 4 次 × 25 条 = Top100
- 关键参数:`sort=downloads` / `dir=desc` / `nonSuspiciousOnly=true` / `numItems=25`
- 触发深度比对的规则:
  - 近期排名提 8 名以上
  - 下载量增长排进前 20
  - 下载和星标都提升 10 名
- 开发历程:浏览器自动化 → 一周三天失败 → 升 GPT 5.5 high 还是高失败率 → 改用 Codex `/goal` 循环 30 次找最少依赖方案 → 第 5 小时发现 Convex 接口

## 设计哲学

- **网页只是表层,API 才是金矿**
- **单日榜单是切片,连续榜单是时间线**
- **不追求找到唯一最优 Skill,只负责把会发光的捞出来**
- **人眼追不上 Skill 生态速度,但 Agent 可以**

## 在本知识库中的角色

- 是 [[Goal-Driven Execution]] 在工程实践中的**活体验证案例**
- 是 [[Skill 榜单情报化]] 概念的代表实现
- 思路可平移到:电商竞品监控、AI 工具趋势追踪、达人榜单等任何"易变公开信息源"
- **直接启发**:Hana 是否有 Skill 市场?能否做"Hana 淘金小镇"?
