# Dreams

> 一句话定义：[[Anthropic]] 推出的异步后台记忆整合机制，灵感来自人类 [[REM 睡眠式记忆整合]]——让 [[Claude]] "白天干活、晚上做梦复盘"，是 [[Maintain 维护]] 的**自动化版本**。

## 基本信息

- **类型**：产品功能 / 后台机制
- **厂商**：[[Anthropic]]
- **状态**：已在 [[Claude Code]] 以「Auto Dream」形式上线；与 [[Memory Files]] 同步灰度
- **首发场合**：Code with Claude 旧金山开发者大会（2026-05 月初）
- **关联实体**：[[Memory Files]]（操作对象）· [[Conway]]（生态拼图）· [[Claude Code]]（首发载体）
- **关联概念**：[[REM 睡眠式记忆整合]] · [[Maintain 维护]] · [[知识复利]]

## 工作机制

当 Claude 智能体在两次工作会话之间「闲下来」时，Dreams 自动启动，对积累的记忆文件做一轮深度整合：

1. **归拢同话题**：把分散在不同文件里的同一话题信息合并到一起
2. **冲突仲裁**：两条记忆冲突 → 保留更新、更可靠的那一条
3. **规律发现**：发现实时对话中人类和 AI 都没注意到的规律

## 触发条件

- 至少累积 5 次对话
- 或距离上次整合 >24 小时
- 或用户手动 `/dream`

## 实战数据（首批接入企业）

Netflix、Rakuten、Wisedocs：

- **首次处理错误率暴降 97%**
- **文档验证提速 30%**

数据印证 Dreams 不是营销话术，而是真实生产力提升。

## 灵感来源

> 人类在快速眼动睡眠阶段，大脑并没有闲着。它在回放白天的经历，强化重要的记忆连接，丢弃无用的噪声信息，并将短期记忆整合为长期记忆。不经历 REM 睡眠的人，记忆力会显著下降。

详见 [[REM 睡眠式记忆整合]]。

## 与本知识库 [[Maintain 维护]] 的对照

| 维度 | [[Maintain 维护]]（本知识库） | Dreams |
|---|---|---|
| 触发 | 用户说"审计 / 体检" | 自动（5 次对话 OR 24h OR 手动 `/dream`） |
| 主体 | [[Hana]] 等 Agent | Claude |
| 操作 | 找矛盾 / 孤儿页 / 缺失反链 / 过时数据 / 隐藏概念 / 空壳页 | 归拢同话题 / 冲突仲裁 / 规律发现 |
| 输出 | `wiki/syntheses/YYYYMMDD-审计报告.md` | 整合后的 Memory Files |
| 冲突处理 | ⚠️ 标矛盾让人裁决 | 保留更新版（自动决策） |

**关键差异**：Dreams 把决策权交给系统，本知识库把决策权留给人。两种哲学各有取舍：Dreams 更省事，本知识库更可控。
