# 规范偏差正常化

> 一句话定义：来自航天工程的概念（挑战者号灾难根因之一）——每次违反规范但没出事，就会降低团队对风险的感知；侥幸累积不等于安全，而是在累积下一次灾难的概率。

## 核心要义

Simon Willison 把这个概念从航天/航空安全领域引入 AI Agent 场景：

每次 Agent 在无人审查下写出正确代码 → "Agent 是可靠的" → 下次更倾向不审查 → 直到某一天 Agent 在关键场景写出致命 bug。

**核心机制**：风险感知不是基于客观风险，而是基于近期个人体验。当"偏差行为"反复无害发生，它会被视为新的正常，而真正的风险阈值在悄悄上移。

## 关键要素

- **侥幸累积**：每次"没事"都让下次更可能"做"，但风险概率未变 (来源: [[../summaries/20260506-SimonWillison-Vibe coding and agentic engineering]])
- **没有反馈机制**：危险悄悄积累，直到事故 (同上)
- **航天起源**：挑战者号 O 形圈在多次低温飞行没出事 → 工程师降低了对低温风险的判断 → 1986 年灾难 (背景知识)
- **AI 场景映射**：Agent "总是写对" → 不再审 → 关键场景翻车 (来源: 同上)

## 与相关概念的关系

- 与 [[Vibe Coding]]：长期 Vibe Coding 实践者最易陷入
- 与 [[Agentic Engineering]]：Willison 承认连他自己也在向这个方向滑
- 与 [[LLM 编程三宗罪]]：三宗罪是要被防范的对象，本概念是为什么会忘记防范
- 与 [[Agent工程化兜底]]：自动化的工程兜底（Hooks / 静态检查）是对抗本概念的解药——把"是否要审"从人的判断变成系统的强制

## 典型应用 / 案例

- Willison 自述：知道 Claude Code 能写对 JSON API → 不再审 → 自我警觉 (来源: [[../summaries/20260506-SimonWillison-Vibe coding and agentic engineering]])
- 航天反例：挑战者号灾难 (背景知识)

## 争议 / 局限

- 完全消除规范偏差代价过高——平衡点在哪？
- 个人警觉难持续，需要外部机制保护
