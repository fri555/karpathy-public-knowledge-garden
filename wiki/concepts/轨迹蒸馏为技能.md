# 轨迹蒸馏为技能

> 一句话定义：轨迹蒸馏为技能是把 Agent 执行过程中的成功路径、失败教训和 workaround，从冗长轨迹压缩成可复用、可迁移、可按需加载的 Skill 文件。

## 核心要义

传统 memory-based agent 往往保存 raw trajectory，再靠检索找相似案例；问题是轨迹长、噪音大、容易占满上下文。轨迹蒸馏为技能的思路相反：不把每次经历都原样塞回上下文，而是从大量经历中抽取反复出现的行为模式、失败边界和操作 SOP，沉淀成声明式 skill。它更像“把事故复盘写进操作手册”，而不是“把监控录像全部存起来”。

## 关键要素

- **成功轨迹提炼**：保留可复用的正确策略和操作顺序 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **失败轨迹提炼**：把失败路径压缩成边界条件、反例和不要做什么 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **并行分析**：让多个 analyst 从不同轨迹批次提出 patch，避免单条轨迹局部过拟合 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **冲突合并**：把 patch 层级合并为统一 skill directory，并做格式/冲突校验 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **迁移验证**：检查 skill 是否能跨模型、跨任务或 OOD 泛化，而不只是记住训练样本。

## 与相关概念的关系

- 与 [[反馈闭环写入文件]]：轨迹蒸馏是更系统化的反馈写入，不只处理用户纠错，也处理 agent 自己的执行历史。
- 与 [[Skills 2.0模块化]]：蒸馏产物通常是 SKILL.md + references/ 的文件夹结构。
- 与 [[上下文膨胀与压缩]]：它把长轨迹压缩成高密度规则，缓解 context bloat。
- 与 [[知识复利]]：每次任务执行都可能反哺技能库，让下一次执行更稳。

## 典型应用 / 案例

- [[../entities/Trace2Skill|Trace2Skill]]：从 execution traces 中并行提 patch，再合并成 conflict-free skill directory (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- SkillRL：把成功/失败轨迹蒸馏进 SkillBank，并在 RL 过程中让 skill library 与 policy 共同演化（来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]]）。

## 争议 / 局限

- 蒸馏质量依赖轨迹多样性；单一场景轨迹容易学到局部 hack。
- 自动 patch skill 有破坏原有稳健性的风险，因此需要训练集验证和 held-out 评估。
