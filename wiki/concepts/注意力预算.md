# 注意力预算

> 一句话定义：LLM 在每次推理时拥有的"注意力总额度"是有限且会被消耗的——每个新 token 都从这个预算里扣分，类似人类的工作记忆容量。

## 核心要义

[[../entities/Anthropic|Anthropic]] 在 [[上下文工程]] 文章中提出的核心隐喻。

来源：Transformer 架构的 n² 注意力机制——n 个 token 产生 n² 对关系。context 越长，模型在每对 token 上能分配的"注意力浓度"越稀薄。

加上模型训练数据中短序列占多数，模型对长序列的处理本就更脆弱。

**实践含义**：不要把"长上下文窗口"当成无限资源——每个 token 都要付出代价，要争取最小高信号集。

## 关键要素

- **架构起源**：Transformer 的 token-to-token attention 计算 (来源: [[../summaries/20260929-Anthropic-Effective context engineering for AI agents]])
- **n² 关系**：context 翻倍则关系数 4 倍 (同上)
- **训练偏差**：短序列在训练中更常见，长序列处理能力天然弱 (同上)
- **预算消耗律**：每个 token 都不是免费的 (同上)

## 与相关概念的关系

- 与 [[上下文腐烂]]：腐烂是预算耗尽的症状
- 与 [[上下文膨胀与压缩]]：压缩是节省预算的手段
- 与 [[Just-in-Time检索]]：按需载入是减少预算消耗的设计
- 与 [[上下文工程]]：本概念是其物理基础

## 典型应用 / 案例

- 决定何时压缩 context：当注意力预算接近耗尽时
- 决定工具集大小：每个工具 schema 都消耗预算
- 决定示例数量：示例是高信号但有成本

## 争议 / 局限

- "预算"是隐喻，没有精确数值——经验性指标
- 不同模型预算"汇率"不同，难以跨模型套用
