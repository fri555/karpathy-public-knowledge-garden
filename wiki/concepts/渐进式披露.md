# 渐进式披露

> 一句话定义：Agent 不一次性加载所有信息，而是通过探索逐层发现相关 context——每次交互产出指导下一步决策的信息（文件大小暗示复杂度，命名暗示用途，时间戳暗示相关性）。

## 核心要义

[[../entities/Anthropic|Anthropic]] 在 [[上下文工程]] 中提出的 Agent 信息发现模式。

与 [[Just-in-Time检索]] 互补：JIT 是"什么时候加载"，渐进披露是"加载多少"——不是全部加载，而是先看元数据/摘要，再按需深入。

类比人类认知：我们不在第一秒就阅读整本书，而是先看目录、再读感兴趣的章节。

## 关键要素

- **探索驱动**：Agent 主动搜索、试探，而非被动接收 (来源: [[../summaries/20260929-Anthropic-Effective context engineering for AI agents]])
- **元数据信号**：文件大小 / 命名 / 目录结构 / 时间戳 (同上)
- **逐层深入**：先粗后细，只在需要时加载更多 (同上)
- **保持精简工作记忆**：只在 context 中保留当前必要信息 (同上)

## 与相关概念的关系

- 与 [[Just-in-Time检索]]：JIT 是时间维（何时），本概念是深度维（多少）
- 与 [[上下文腐烂]]：渐进披露减少 context 中的 token 量，减缓腐烂
- 与 [[上下文工程]]：本概念是 context engineering 的信息发现策略
- 与 [[三层架构]]：raw/ → summaries/ → entities/concepts/ 的分层 = 渐进披露的文件系统实现

## 典型应用 / 案例

- Claude Code 用 glob 扫描目录 → grep 搜索关键词 → read 读取具体文件 (来源: [[../summaries/20260929-Anthropic-Effective context engineering for AI agents]])
- 本库 Agent 查询流程：先读 index.md → 定位相关页 → 读具体页 (来源: 自我观察)

## 争议 / 局限

- 探索可能走弯路，浪费 [[注意力预算]]
- 需要好的工具和启发式引导
