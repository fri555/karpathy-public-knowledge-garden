# 记忆文件 vs 技能文件

> 一句话定义：Agent 的持久化存储必须把**偏好（记忆文件）**和**操作 SOP（技能文件）**拆开——前者是"用户喜欢什么/不喜欢什么"，后者是"这件事该怎么做"。混在一起会导致规则互相打架。

## 核心要义

两个独立团队（[[Shubham Saboo]] 和 [[Shareuhack]]）基于实战独立收敛到同一套二分法：

- **记忆文件 / Memory / Auto Memory**：用户偏好、BAD/GOOD 样本、debug 心得——**谁写的？人 + AI 都可能**
- **技能文件 / Skills / SOP**：操作规则、任务说明书、工作流步骤——**谁写的？人定义**

Saboo 的原始发现：最初两周把所有东西塞进同一个文件（偏好+规则+经验+教训），Agent 加载到的上下文经常互相打架（第一周形成一种偏好，第二周又写入更明确规则，两者冲突）。拆开后技能文件"积累更快、效果更直接"。

Shareuhack 的等价表达：CLAUDE.md = 永久性规则（人写），Skills = 可执行工作流（人定义），Auto Memory = Claude 自己的学习笔记——**三者各有不该放的东西**（见 [[CLAUDE.md四层架构]]）。

## 关键要素

- **记忆文件特征**：偏好型、样本型、可被覆盖（来源: [[20260528-36氪-Saboo30天6Agent复盘]]）
- **技能文件特征**：指令型、SOP 型、不被日常反馈覆盖（来源: [[20260528-36氪-Saboo30天6Agent复盘]]）
- **拆分线**：能写成"下次请按 Y 来做"的 → 技能文件；只能写成"我喜欢/不喜欢 X"的 → 记忆文件

## 与相关概念的关系

- 与 [[CLAUDE.md四层架构]]：Saboo 的"记忆 vs 技能"二分是 Shareuhack 四层架构的子集
- 与 [[三套记忆系统]]：Saboo 的记忆文件 ≈ Auto Memory，技能文件 ≈ Skills
- 与 [[上下文膨胀与压缩]]：拆开文件是防止膨胀的手段之一

## 典型应用 / 案例

- Kelly 的 BAD/GOOD 文件（记忆） + "推文格式 SOP"（技能） (来源: [[20260528-36氪-Saboo30天6Agent复盘]])
- 本知识库：AGENT.md（规则/技能） + log.md + summaries/（记忆/经验）——天然符合这个二分法 (来源: 自我观察)

## 争议 / 局限

- "偏好"和"规则"的边界不总是清晰——比如"不要用 bullet points"是偏好还是规则？
- 技能文件过多时需要 [[Skills 2.0模块化]] 管理
