# 30/70 定律

> 一句话定义：**Agent 的"智能"部分只占 30% 的工作量，剩下 70% 全在"连接"上**——管权限、防泄露、协议互通——这是 Agent 从 demo 走向生产的核心工程现实。

## 核心要义

[[20260528-掘金-MCP协议落地三条铁律|沙河 rain]] 基于半年企业级 Agent 实战提出的金句。这条定律颠覆了"AI 进步 = 模型进步"的直觉：

- 30%：选模型、写 prompt、调参——这是大家都在讨论的
- 70%：权限管理、协议互通、容错兜底、监控告警——这是大家不愿讨论的

含义：**很多 Agent 项目失败不是因为模型不够强，而是因为低估了连接成本**。

## 关键要素

- **权限**：字段级权限 + 最小权限令牌（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **通信**：[[Agent Gateway协议翻译层]]（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **容错**：[[工具调用幻觉]] 三层防护（来源: [[20260528-掘金-MCP协议落地三条铁律]]）

## 与相关概念的关系

- 与 [[LLM 编程三宗罪]]：三宗罪讲模型本身缺陷，30/70 讲模型之外的工程负担
- 与 [[Hooks确定性执行]]：Hooks 也是"工程兜底"哲学的兑现
- 与 [[Goal-Driven Execution]]：GDE 是 30% 智能的最优实践，30/70 提醒还有 70% 在等着
- **跨域种子**：电商 AI 运营中"模型选型" vs "ERP/客服/SKU 接入"也是 30/70 关系——直接对应先生本职位 ⭐

## 典型应用 / 案例

- 沙河 rain 两个企业级 Agent 项目的实战 (来源: [[20260528-掘金-MCP协议落地三条铁律]])

## 争议 / 局限

- 30/70 是经验值，不同行业/场景的比例可能不同（C 端可能 50/50，B 端可能 20/80）
- 不适用于纯文本生成场景（连接成本接近零）
