# Ingest 录入

> 一句话定义：[[LLM Wiki]] 三个核心操作之一——把一份新资料编译进 Wiki，通常牵动 10–15 个页面更新。

## 核心要义

不是简单的"上传索引"，而是**让 Agent 把新资料吃透并融入已有知识网络**。

## 标准流程（按 [[../../AGENT|AGENT.md]]）

1. 原始文件落到 `raw/`，命名 `YYYYMMDD-来源-标题.ext`
2. Agent 读一遍 + 与人讨论要点（确认核心，不脑补）
3. 在 `wiki/summaries/` 写摘要页
4. 抽出新出现的 [[实体页|实体]] 和 [[概念页|概念]]，建/更新对应页
5. 建立交叉引用（所有相关页 `[[双向链]]`）
6. 更新 `wiki/index.md`
7. 追加 `wiki/log.md`
8. 矛盾点用 `> ⚠️ 矛盾：...` 块标注，等待人类裁决

## 关键经验法则

- **一份资料 = 5–15 个 Wiki 页更新**。如果只动 1–2 个，要么资料太薄，要么没做透
- 已存在的页**追加新信息并标注来源**，不要覆盖
- Karpathy 喜欢与 LLM 讨论要点后再写——避免 LLM 自说自话

## 与相关概念的关系

- 是 [[LLM Wiki]] 的入口操作
- 输出物喂给 [[Query 查询]] 使用
- 累积到一定量触发 [[Maintain 维护]]
