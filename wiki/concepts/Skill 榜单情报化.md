# Skill 榜单情报化

> 一句话定义:把"每天都会变的公开信息源(榜单/排行/列表)"通过 Agent **每日固定路线抓取 + 跨日对比**,变成可被复盘的**时间线情报网**——单日榜单只是切片,连续榜单才是信号。

## 核心要义

人类看榜单只能看到"今天的快照",但 Agent 连续运行后能看到:
- 谁是新来的
- 谁掉下去了
- 谁突然多人开始用
- 谁连续几天都在涨
- 哪个老牌因版本更新重新崛起

**这些是网页本身永远不会告诉你的**——榜单只展示瞬时排名,**时间维度上的变化才是真情报**。

## 关键要素 (来源: [[20260527-知乎-卡尔-淘金小镇Skill榜单情报项目]])

1. **固定路线**:每天去同一个地方,拿同一类数据
2. **固定时间**:做到时间可比
3. **多日对比报告**:不是看快照,是看 diff
4. **触发规则**:不是全部告诉你,只挑"会发光的"——新进 / 突涨 / 连涨 / 版本回春
5. **复刻用户视角**:用 API 而非 DOM 抓,才接近用户实际看到的

## 与相关概念的关系

- **核心实现**:[[淘金小镇]] 是这一概念的代表实现
- **底层支撑**:依赖 [[Goal-Driven Execution]](让 Agent 自主循环到拿到数据)
- **数据获取**:依赖 [[复刻用户视角拿数据]](API > HTML)
- **本质**:[[知识复利]] 在情报领域的应用——单日没价值,连续积累就涌现价值
- **可平移领域**:
  - 电商竞品 SKU 榜单(对应先生 AI 运营工作)
  - GitHub trending
  - HuggingFace 模型榜
  - AppStore / Steam 榜
  - 小红书 / B 站爆款榜
  - 知乎/微博热榜

## 典型应用 / 案例

- [[淘金小镇]] —— ClawHub Skill Top100 每日抓取 + 多日对比 (来源: [[20260527-知乎-卡尔-淘金小镇Skill榜单情报项目]])
- **可启发先生**:
  - "小红书运动户外类目每日热门 SKU"情报站
  - "AI 工具圈每日新晋"情报站(支撑"慢思考研究所"内容输出)

## 争议 / 局限

- 信号识别规则(如"近期排名提 8 名以上")需要经验,初期可能漏报或误报
- 数据源 API 一旦封禁,整个情报网就断了——需要冗余源
- 长时间运行成本(token + 存储)需要测算
- 反过来:如果太多 Agent 都做榜单情报化,**榜单本身可能被刷扰**
