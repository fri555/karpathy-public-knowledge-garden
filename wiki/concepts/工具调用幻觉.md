# 工具调用幻觉

> 一句话定义：LLM 不只编造文本，还会在 Agent 场景中编造**不存在的工具名、错误的参数格式、无限循环调用**——这是比文本幻觉更危险的新型缺陷，必须用工程手段三层防护。

## 核心要义

[[20260528-掘金-MCP协议落地三条铁律|沙河 rain]] 提出的 Agent 时代新型缺陷。三种典型表现：
1. 模型"想象"出一个不存在的工具名 → 调用失败
2. 模型给工具传了错误的参数格式 → 服务端报错
3. 模型在循环调用中"卡住" → 反复调用同一个工具出不来

为什么危险：**在开发环境很难复现**（开发时模型/工具/prompt 固定），一到生产用户输入千变万化，问题就爆发。

## 三层防护

| 层 | 名称 | 机制 |
|---|---|---|
| 1 | **Schema 校验** | MCP Server 收到调用前 JSON Schema 校验参数格式 |
| 2 | **工具存在性校验** | Agent planning 阶段维护实时白名单，不存在的 tool name 触发 fallback |
| 3 | **循环检测** | 同一工具连续调用 N 次（如 3 次）且参数无变化 → 强制中断转人工 |

## 关键要素

- **Schema 校验前置**：不让脏请求打到后端服务（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **白名单实时维护**：planning 阶段就拦截（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **循环参数比较**：参数无实质变化才算循环（来源: [[20260528-掘金-MCP协议落地三条铁律]]）

## 与相关概念的关系
