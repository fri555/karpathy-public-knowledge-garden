# Goal 模式自动化爬虫

> 一句话定义:面对**易变/反爬/无 API**的目标站点,**不要写 prompt-heavy 的硬规则脚本**,而是用 [[Codex]] 等 Agent 的 `/goal` 无限循环模式——给定**强成功标准**,让 Agent 自主迭代到达成。是 [[Goal-Driven Execution]] 原则在爬虫工程中的具体落地。

## 核心要义

传统爬虫思路:
- 写超详细 prompt(浏览器路径 / DOM 选择器 / 异常处理 / 重试逻辑)
- 然后听天由命

**[[卡尔的AI沃茨]] 实测的失败案例** (来源: [[20260527-知乎-卡尔-淘金小镇Skill榜单情报项目]]):
- 浏览器自动化 + 复杂提示语 → **一周三天失败**
- 升级到 GPT 5.5 high → 失败率还是超高
- **改用 Codex `/goal` 循环 30 次**,目标设为"找出依赖最少的方案" → 第 5 小时发现 [[Convex]] API,问题解决

## 关键要素

1. **目标必须强**(参考 [[Goal-Driven Execution]]):不是"爬下来",而是"找出依赖最少的方案,循环 30 次都能拿到 Top100"
2. **循环数足够**:30 次给 Agent 足够探索空间
3. **失败可重连**:断掉再开,Agent 接续探索
4. **允许 Agent 发挥想象力**:不锁死路径,让它自己发现 API/Convex 入口
5. **prompt 要精简**:复杂提示语反而限制 Agent 探索

## 与相关概念的关系

- **直接落地**:[[Goal-Driven Execution]] 在爬虫领域的具体形态
- **典型代表**:[[淘金小镇]] —— 最高赞案例
- **配合**:[[复刻用户视角拿数据]] —— Agent 探索时优先找 API
- **反对**:Prompt-Heavy 思路(详细规则 + 高级模型 ≠ 高成功率)
- **本知识库的潜在应用**:ingest 时若网页反爬,可以让 Agent 自主探索 → 而非每次靠人指挥换抓法

## 典型应用 / 案例

- [[淘金小镇]] 攻克 ClawHub 反爬 → 发现 Convex API → 稳定运行 (来源: [[20260527-知乎-卡尔-淘金小镇Skill榜单情报项目]])
- **可应用场景**:
  - 知乎反爬时,让 Agent 自主探索 API/web/移动端等替代路径
  - 飞书/钉钉等企业 SaaS 的数据导出自动化
  - 电商平台数据采集(运营常用)

## 争议 / 局限

- **token 成本高**:循环 30 次比单次 prompt 贵 30 倍
- **不可预期**:目标设不强 Agent 会乱跑
- **不适合简单任务**:5 分钟能写完的脚本不用上 Goal 模式
- **法律 ToS**:Agent 可能"创造性"绕过限制,带来合规风险——需要明确边界
- **平台变更**:Agent 发现的 API 平台会改,Goal 模式需要重新跑
