# 多路并发竞优

> 一句话定义：多路并发竞优是同时让多个模型、Prompt 或策略生成候选方案，再通过评测选择最优解，用并行探索对抗 LLM 输出随机性。

## 核心要义

LLM 生成常有随机性，单一路径失败不代表任务不可做。多路并发竞优把“串行反复追问”改成“一次启动多个独立候选”，再用测试与评分筛选。它本质是把计算成本换成时间和稳定性，适合候选可并行、结果可评测的任务。

## 关键要素

- **不同 creator 路径**：例如 Anthropic skill-creator、OpenClaw skill-creator、Superpowers writing-skills (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **不同模型 / Prompt**：利用模型和提示模板差异扩大搜索空间 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **First-Time Pass Rate**：目标是提高首次可用候选出现概率，减少用户等待 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- **统一评测门禁**：并发只负责生成多样性，最终仍靠测试和评分收敛。

## 与相关概念的关系

- 与 [[评测驱动Skill生成]]：没有评测，多路并发只会制造更多垃圾候选。
- 与 [[Subagents上下文隔离]]：实现上常用子 Agent 隔离不同候选路线。
- 与 [[Agent团队三阶段曲线]]：早期混乱期可用多路探索找到可行路径，稳定期再收敛为单一路线。

## 典型应用 / 案例

- [[../entities/SkillFactory|SkillFactory]] 并行调用 3 种 creator 策略生成 skill 候选 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。
- [[../entities/Trace2Skill|Trace2Skill]] 并行派发 analyst sub-agents 对不同轨迹批次提出 patch，再合并 (来源: [[../summaries/20260608-知乎-SkillFactory面向Harness设计的技能工厂]])。

## 争议 / 局限

- 成本更高；如果任务不可自动评测，会把选择压力转移给人。
- 多个候选可能各有局部优点，合并过程需要冲突检测。
