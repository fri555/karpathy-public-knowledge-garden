# 字段级权限

> 一句话定义：MCP Server 不应只给 Agent "能用/不能用"的开关，而要让每个工具声明**数据敏感度等级**，给每个会话绑定**最小权限令牌**——权限检查发生在模型"决定做什么"之前，而不是"已经做了什么"之后。

## 核心要义

[[20260528-掘金-MCP协议落地三条铁律|沙河 rain]] 总结的 MCP 第一坑解药。原始问题：暴露了 `search_docs` 和 `send_email` 两个工具，模型可能"自作主张"调用 `send_email` 把敏感内容发出去。

字段级权限的两个核心动作：
1. **声明等级**：每个工具在 `tools/list` 响应里加 `required_permission_level`（公开/内部/机密）
2. **前置检查**：在 Agent 的 planning 阶段就拒绝，不等 execution 阶段才报错

**范式转变**：从"模型决定 → 系统拦截"变成"系统过滤 → 模型选择"。

## 关键要素

- **数据敏感度等级**：公开 / 内部 / 机密（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **会话级最小权限令牌**：不同用户/场景的会话拿到的工具池不同（来源: [[20260528-掘金-MCP协议落地三条铁律]]）
- **planning 阶段检查**：而不是 execution 阶段（来源: [[20260528-掘金-MCP协议落地三条铁律]]）

## 与相关概念的关系

- 与 [[MCP]]：是 MCP Server 安全实践的核心
- 与 [[Hooks确定性执行]]：都是"工程化兜底"哲学的兑现——给 LLM 限定可选项
- 与 [[Agent工程化兜底]]：字段级权限是兜底的"前置"形态

## 典型应用 / 案例

- 企业内部文档查询 Agent 防止误发邮件 (来源: [[20260528-掘金-MCP协议落地三条铁律]])
