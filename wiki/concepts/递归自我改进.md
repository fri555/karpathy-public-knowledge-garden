# 递归自我改进

> 一句话定义：AI 系统**用自己改进自己**的循环范式——通过运行结果反馈、自动调整自身（模型权重 / Prompt / 工具调用策略），实现性能随时间累积的复利增长。Karpathy 2026-05 加入 [[Anthropic]] 预训练团队后，被推测的重点研究方向之一。

## 核心要义

传统 AI 改进路径：**人工**收集数据 → **人工**标注 → **人工**训练 → **人工**评估 → 迭代。
递归自我改进路径：**AI 自己**生成训练数据 → **AI 自己**评估 → **AI 自己**修正 → 进入下一轮循环。

关键差异是「人工」环节的去除程度：
- **半自动**：人定义任务和评估标准，AI 自我训练（现状大多数）
- **全自动**：AI 自己定义任务、自己评估、自己改（前沿研究）

## 关键要素

- **自评估能力**：AI 能判断自己输出的好坏（不依赖人）
- **闭环反馈**：评估结果直接回流影响下一次训练/推理
- **可复利的改进信号**：每一轮改进都"真的"让系统变好（避免幻觉式自我提升）

## 在本知识库中的角色

- 是 [[Anthropic]] 招募 [[Andrej Karpathy]] 进入预训练团队后，外界推测他可能聚焦的方向
- 与 [[知识复利]] 同构：每一次操作都让系统更好
- 与 [[LLM Wiki]] 的 [[Maintain 维护]] 有概念呼应：让系统在维护中自我升级

## 风险与争议

- **奖励黑客（reward hacking）**：AI 可能学到"看起来更好"而不是"真的更好"
- **错误放大**：错的信号被自己采用后形成偏差累积
- **可控性下降**：闭环越紧，人介入修正的窗口越小
