# Agent 团队三阶段曲线

> 一句话定义：每个 Agent 团队都会经历 **混乱期（1-7 天，负收益）→ 稳定期（8-21 天，少量编辑）→ 复利期（21 天+，自动跑通）** 三阶段——**绝大多数人死在第 1 阶段**。

## 核心要义

[[Shubham Saboo]] 在 30 天 6 Agent 复盘中总结的实战曲线，是养 Agent 的"心理预期管理工具"。

| 阶段 | 时间 | 体感 | 风险 |
|---|---|---|---|
| **混乱期** | 第 1-7 天 | 修改成本 > 人工完成成本 | 大部分人在这里放弃 |
| **稳定期** | 第 8-21 天 | 输出可用，需少量编辑 | 反馈不沉淀就停滞 |
| **复利期** | 第 22 天+ | 自动跑通，效率反超人工 | 上下文膨胀拖慢 |

## 关键要素

- **混乱期负收益是必然的**——不是模型不行，是规则系统还没建起来（来源: [[20260528-36氪-Saboo30天6Agent复盘]]）
- **从混乱期出来的唯一办法**：[[反馈闭环写入文件]]——把每次修改都沉淀到文件
- **复利期的标志**：新任务能继承过去的经验，不再从零开始
- **能否进入复利期 ≈ 能否克制"上得太快"的冲动**——Saboo 自己也犯了两周搭 6 个 Agent 的错（来源: [[20260528-36氪-Saboo30天6Agent复盘]]）

## 与相关概念的关系

- 与 [[纠错式Prompt工程]]：三阶段是结果，纠错式是过程
- 与 [[Goal-Driven Execution]]：第 4 周复利期 = Karpathy 说的"给它成功标准，看着它完成"的实证
- 与 [[知识复利]]：本库的复利原则跟 Agent 团队复利期完全同源——每加一份资料都让系统更好

## 典型应用 / 案例

- [[Mac Mini Agent Fleet]] 完整经历了三阶段 (来源: [[20260528-36氪-Saboo30天6Agent复盘]])
- 本知识库 2026-05-24 启动至今约 5 天，刚出"混乱期"边缘 (来源: 自我观察)

## 争议 / 局限

- 时间分段（1-7/8-21/21+）是 Saboo 单一案例的经验值，不一定普适
- 单 Agent 和多 Agent 的曲线可能差异巨大
