# 三套记忆系统

> 一句话定义：Claude Code 有三套**完全不同**的记忆机制——CLAUDE.md（人写的规则）/ Auto Memory（Claude 自己写的笔记）/ Session Memory（系统写的状态摘要）——**混用是 Agent 行为不稳定的最常见原因**。

## 核心要义

[[Shareuhack]] 提出的关键澄清。三套系统的本质差异：

| 系统 | 谁写的 | 载入时机 | 适合放什么 |
|---|---|---|---|
| **CLAUDE.md** | 你（人类） | 每 session 全量载入 | 永久性规则、禁止事项、架构规范 |
| **Auto Memory** | Claude 自己 | 跨 session 累积载入 | 工作笔记、debug 心得、偏好学习 |
| **Session Memory** | 系统自动 | 对话延续时载入 | "上次做了什么"的摘要 |

## 关键要素

- **常见错误 1**：把规则放进 Auto Memory——优先级低于 CLAUDE.md，且 Claude 可能修改自己的笔记导致规则"漂移"（来源: [[20260528-Shareuhack-CLAUDE四层架构实战]]）
- **常见错误 2**：把工作状态放进 CLAUDE.md——每 session 都会载入过期状态，浪费 context（来源: [[20260528-Shareuhack-CLAUDE四层架构实战]]）
- **常见错误 3**：不知道 Auto Memory 存在——它默默累积在 ~/.claude/ 目录下的 memory 文件中（来源: [[20260528-Shareuhack-CLAUDE四层架构实战]]）

## 与相关概念的关系

- 与 [[CLAUDE.md四层架构]]：三套记忆是四层中"Memory 层"的内部细分
- 与 [[记忆文件vs技能文件]]：Saboo 的二分法对应 Auto Memory（记忆）vs CLAUDE.md/Skills（技能/规则）
- 与 [[Memory Files]]（Anthropic 产品）：Anthropic 把 Memory 做成产品级，三套记忆是底层机制
- 与 [[REM 睡眠式记忆整合]]：Dreams 是对 Memory 的自动整合

## 典型应用 / 案例

- [[Shareuhack]] fleet 严格区分三套系统，是其稳定性的关键 (来源: [[20260528-Shareuhack-CLAUDE四层架构实战]])

## 争议 / 局限

- 普通用户可能不需要懂这么多层——简单项目用好 CLAUDE.md 就够
- Auto Memory 是 Claude 自管，可能有不可预测的内容污染
