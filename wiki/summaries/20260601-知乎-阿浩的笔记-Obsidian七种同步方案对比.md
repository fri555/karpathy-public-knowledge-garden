# 20260601 · 知乎 · Obsidian 七种主流同步方案对比

> 一句话总结：Obsidian 用户从轻度到重度的全谱系同步方案指南——没有"最好"的方案，只有"最适合"的组合。作者以实战测评视角对比了 7 种方案，核心建议是**按使用场景分库**（主库 Git 版本控制 + 移动端轻量同步）。

## 关键收获

### 1. 同步不是技术问题，是取舍问题

- 速度 vs 可靠性：坚果云快但 503（请求频率限制）
- 隐私 vs 便利：Syncthing 全端需同时在线
- 历史记录 vs 实时性：Git 可回滚但非实时

### 2. "双库结构"是核心方法论

作者推荐的**主库 + 移动库分离**：
- 主库：Git 版本控制，深度写作
- 移动库：坚果云轻量同步，灵感记录 → 定期归档到主库

与先生知识库的 [[Maintain 维护]] 理念同构——不同层的操作有不同的同步策略。

### 3. 作者的写作闭环

S3 存储桶（笔记 + 图床分离）→ PicGo 图床 → images Auto Upload 自动上传 → mp preview 公众号排版。这个"写作→发布"的链路对先生可能的公众号出口有参考价值。
