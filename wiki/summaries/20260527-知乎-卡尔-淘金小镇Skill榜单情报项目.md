# 20260527 · 知乎 · 卡尔 · 淘金小镇:Skill 榜单情报项目

> 一句话摘要:作者卡尔做了一个叫 **[[淘金小镇]]**(skillrush-town)的 Skill,每天爬 **[[ClawHub]]** 下载榜 Top100,**把"易变的公开信息源"变成"可被 Agent 复盘的情报网"** —— 这是 [[Goal-Driven Execution]] 和 [[知识复利]] 在 Skill 生态情报领域的精妙落地。

## 核心论点

1. **榜单 = 时间线**:单日榜单只是切片,但**连续多日榜单**就变成了时间线数据,可识别"新进/掉榜/突涨/连涨"等真正有信号的事件——这是网页本身永远不会告诉你的。
2. **Goal-Driven 战胜 Prompt-Heavy**:作者用 Codex 的 `/goal` 无限循环模式(目标:"找出依赖最少的方案,循环 30 次都能拿到 Top100"),**比写十几条限制的提示语更有效**——直接呼应 [[CLAUDE.md]] 第 4 条原则。
3. **复刻用户视角拿数据**:不是抓 HTML 文本,而是**找网页自己用的 API**(Convex `skills:listPublicPageV4`),这样拿到的数据更接近用户实际看到的。
4. **人眼追不上 Skill 生态的速度,但 Agent 可以**:Skill 生态像早期 Chrome 插件市场,每天上传,信息流太快;**Agent 用固定路线 + 每日比较** 可以把"会发光的"筛出来。

- **技术栈**:Codex(`/goal` 模式)→ 发现 [[Convex]] API `skills:listPublicPageV4` → cursor 分页拿 4×25=100 条
- **关键技术发现**:`clawhub.ai/skills?sort=downloads` 背后是 Convex 后端,直接调 API 比浏览器自动化稳定 10 倍
- **追加规则**:近期排名提 8 名以上 / 下载量增长前 20 / 下载+星标双升 10 名 → 触发深度比对
