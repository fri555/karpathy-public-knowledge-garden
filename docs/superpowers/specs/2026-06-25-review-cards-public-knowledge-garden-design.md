# 公开知识花园复习卡片设计

## 背景

这个知识库正在从 AI/Agent 专题库，升级成个人认知操作系统：只要某份知识能帮助长期判断、学习、产品思考、工程能力、商业理解或认知框架，它就可以进入知识库。AI 和 Agent 仍然是重要主线，但不能变成信息茧房。

当前仓库已经有一套稳定的 LLM Wiki 结构：

- `raw/` 保存原始资料快照，作为私有证据层和后续再处理材料。
- `wiki/summaries/` 保存单份资料摘要。
- `wiki/entities/` 保存人物、组织、产品和项目。
- `wiki/concepts/` 保存可复用的概念和方法。
- `wiki/syntheses/` 保存跨资料综述和二次分析。
- `wiki/index.md` 和 `wiki/log.md` 提供导航与操作审计。

现在缺少的是主动消化层：读者需要一个公开、手机友好的方式，把知识重新变成问题来复习，而不只是阅读页面。

## 产品定位

建设一个公开的个人知识花园，并提供复习卡片模式。

第一版不是完整 Anki，也不是私密版 NotebookLM。它要让知识变得：

- 可阅读：像公开知识花园一样浏览。
- 可回忆：通过卡片把“看过”变成“能想起来”。
- 可理解：通过解释和相关概念帮助消化。
- 可追溯：能回到 wiki 页面和原始来源链接。

第一版不需要登录、个人进度、数据库或真正的间隔重复算法。

## 目标

- 将现有 wiki 发布成手机友好的公网网站。
- 从现有 wiki 页面生成复习卡片。
- 每日 ingest 后自动生成新卡片。
- 让读者从首页进入轻量复习会话。
- 让读者可以针对某个 wiki 页面进行复习。
- 保留来源可追溯性，但不公开转载第三方 `raw/` 全文快照。

## 非目标

- 第一版不做用户账号。
- 第一版不保存个人学习进度。
- 第一版不做真正的间隔重复算法。
- 第一版不公开展示第三方 `raw/` 全文快照。
- 不替代本地 Markdown wiki；本地 wiki 仍然是知识源头。

## 公开边界

公网网站可以公开：

- 渲染后的 `wiki/concepts`、`wiki/entities`、`wiki/summaries`、`wiki/syntheses` 页面。
- 复习卡片。
- `raw` 元数据，例如标题、来源、日期、原链接、信源层级、入库理由。
- 原始来源链接。

公网网站不能公开：

- 第三方文章的完整 `raw` 快照。
- 不准备公开的私人笔记。
- 敏感本地文件路径。
- 标记为私密、未确认或不适合公开的材料。

这个边界能让网站保持有用和可追溯，同时避免变成转载站。

## 信息架构

### 首页

首页优先服务手机端行动：

- 主操作：开始今日复习包。
- 最新 ingest。
- 最近更新概念。
- 学习路径入口。
- 概念、摘要、综述、实体入口。

### 知识花园

知识花园负责渲染公开 wiki。每个页面应包含：

- 标题和元数据。
- 页面正文。
- 反链和相关概念。
- 来源链接。
- “复习此页”入口。

### 卡片模式

卡片模式一次展示一张卡：

- 先展示问题。
- 答案默认折叠。
- 展开后展示答案和解释。
- 提供来源 wiki 页面链接。
- 有原始来源时提供原始来源链接。
- 展示相关概念。
- 提供下一张操作。

第一版只保留当前会话状态在内存中。刷新页面后可以重置当前复习会话。

### 今日复习包

每日 ingest 后生成或刷新一个公开复习包：

- 来自新增页面的卡片。
- 从相关旧页面中抽取的高价值卡片。
- 少量重点概念。
- 几个反常识题或应用题。

如果当天没有合格 ingest，可以从已有高价值页面中抽一组旧知识复习包。

### 来源元数据页

`raw` 元数据页只展示：

- 来源标题。
- 来源类型和信源层级。
- 发布或抓取日期。
- 原始 URL。
- 对应 summary 页面。
- 入库理由。
- 公开引用说明。

## 卡片类型

第一版支持五类卡片。

### 定义卡

目的：防止重要术语只留下模糊印象。

形态：

- 问题：这个概念是什么意思？
- 答案：一句简明定义。
- 解释：它为什么重要，以及容易和什么混淆。

### 反常识卡

目的：保留一份资料带来的 surprise 或信念更新。

形态：

- 问题：这份资料挑战了哪个直觉？
- 答案：修正后的判断。
- 解释：这个判断背后的理由或证据。

### 对比卡

目的：区分相近概念。

形态：

- 问题：A 和 B 的关键差异是什么？
- 答案：最短但有用的区别。
- 解释：分别适用于什么场景。

### 应用卡

目的：把知识连接到判断和行动。

形态：

- 问题：这个知识如何用于产品、工程、学习或判断？
- 答案：一个具体应用。
- 解释：约束、风险或例子。

### 连接卡

目的：让知识网络保持活性。

形态：

- 问题：这个知识和哪些既有概念发生连接？
- 答案：一组相关概念。
- 解释：这些连接为什么重要。

## 卡片数据模型

第一版可以把卡片生成为静态 JSON。

```json
{
  "id": "concept-llm-wiki-definition-001",
  "type": "definition",
  "question": "什么是 LLM Wiki？",
  "answer": "LLM Wiki 是由 LLM Agent 持续维护和编译的知识库，它把 raw、summary、concept、entity 和 synthesis 组织成可复用的知识网络。",
  "explanation": "重点不是存文档，而是让每份资料和每次提问都沉淀为可链接、可复用、可继续生长的知识。",
  "sourcePage": "wiki/concepts/LLM Wiki.md",
  "sourceUrl": "https://example.com/original-source",
  "concepts": ["LLM Wiki", "知识复利", "Ingest 录入"],
  "difficulty": "medium",
  "created": "2026-06-25"
}
```

必填字段：

- `id`
- `type`
- `question`
- `answer`
- `sourcePage`
- `created`

建议字段：

- `explanation`
- `sourceUrl`
- `concepts`
- `difficulty`
- `tags`
- `sourceTier`
- `ingestDate`

## 生成流程

Markdown 仍然是唯一知识源头。

```text
wiki 页面
  -> 页面解析器
  -> 卡片生成器
  -> 复习包生成器
  -> 静态 JSON
  -> 公网前端
```

生成器优先利用现有页面结构：

- 概念页的一句话定义和核心要义。
- 摘要页的核心论点。
- 综述页的结论和对比。
- 实体页的关系描述。
- `待深入` 和 `相关资料` 区块。

后续 ingest 可以增加可选的 `复习卡片候选` 区块，以提升卡片质量。但第一版不能要求重写所有旧页面。

## 每日 Ingest 集成

每次每日 ingest 完成后：

1. 从 `wiki/log.md` 或 git diff 识别新增/更新的 wiki 页面。
2. 为这些页面生成卡片。
3. 将卡片链接到来源 wiki 页面和原始 URL。
4. 把精选卡片加入今日复习包。
5. 重建公网 JSON。

每日自动化应使用开放知识策略：

- AI 和 Agent 重要，但不是唯一主题。
- 允许高质量跨域知识进入。
- 跨域资料必须记录入库理由。
- 低密度新闻、营销稿和泛泛趋势文仍然拒收。

## 前端行为

前端优先静态化。

核心页面：

- 首页。
- wiki 页面。
- 卡片复习页。
- 今日复习包页。
- 来源元数据页。

核心交互：

- 展开答案。
- 下一张卡片。
- 按卡片类型筛选。
- 按概念或学习路径筛选。
- 跳转到来源 wiki 页面。
- 跳转到原始来源 URL。

第一版不需要服务端会话状态。

## 部署形态

第一版可以部署成静态站，目标平台可选 Vercel、Cloudflare Pages 或 GitHub Pages。

推荐架构：

- 本地 Markdown wiki 仍然是 canonical source。
- 构建脚本负责渲染公开页面和 JSON。
- 静态前端消费生成后的公开资产。
- 部署时只发布 public output 目录。

## 风险与缓解

### 版权风险

风险：公开第三方文章全文快照，可能接近转载。

缓解：只公开 `raw` 元数据和原始链接，不公开第三方 `raw` 全文。

### 卡片质量风险

风险：自动生成卡片可能退化成浅层定义题。

缓解：优先生成反常识卡、对比卡、应用卡、连接卡，而不仅是定义卡。

### 知识漂移风险

风险：开放选题后，知识库可能变成杂货铺。

缓解：要求记录入库理由，并用学习路径组织跨域知识。

### 过度建设风险

风险：过早加入账号、进度、间隔重复，会拖慢第一版上线。

缓解：第一版保持公开、无状态。个人进度可以后续再加。

## 验收标准

- 公网手机端网站可以渲染 wiki 页面。
- 首页有“开始复习”路径。
- 卡片会话可以展示问题、展开答案、进入下一张。
- 卡片能回链到来源 wiki 页面。
- 来源页只公开元数据和原链接，不公开第三方 `raw` 全文快照。
- 现有 wiki 页面可以生成第一批卡片。
- 每日 ingest 后可以刷新卡片数据和今日复习包。
