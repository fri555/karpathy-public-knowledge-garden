# 公开知识花园复习卡片需求文档

## 1. 项目概述

### 1.1 项目名称

公开知识花园复习卡片

### 1.2 项目定位

本项目是个人认知操作系统的公网学习入口：将现有 Karpathy 式知识库发布为公开知识花园，并在其上提供轻量复习卡片模式。

第一版重点解决两个问题：

- 防遗忘：把“看过的知识”变成可以反复回忆的问题。
- 促理解：通过解释、对比、应用和连接，让知识从静态页面变成可消化材料。

第一版不是完整 Anki，不做账号和个人学习进度；也不是私密 NotebookLM，不做登录后的私人问答空间。

### 1.3 背景

当前知识库已经形成 LLM Wiki 结构：

- `raw/`：原始资料私有证据层。
- `wiki/summaries/`：单份资料摘要。
- `wiki/entities/`：实体页。
- `wiki/concepts/`：概念页。
- `wiki/syntheses/`：跨资料综述。
- `wiki/index.md`：总索引。
- `wiki/log.md`：操作日志。

现有问题是知识已经可以持续进入和整理，但主动复习与理解训练不足。知识被写进 wiki 后，如果没有卡片、提问和复习入口，仍然容易变成“死知识”。

## 2. 用户与使用场景

### 2.1 目标用户

主要用户是知识库所有者本人，同时允许公网读者浏览和学习。

### 2.2 核心场景

1. 手机上打开网站，直接开始今日复习。
2. 浏览某个概念页时，点击“复习此页”，进入围绕该页面生成的卡片。
3. 每日 ingest 后，系统自动生成今日复习包。
4. 阅读一张卡片后，可以跳回对应 wiki 页面继续深读。
5. 查看某份资料的来源元数据和原始链接，但不直接阅读第三方 raw 全文快照。

### 2.3 用户收益

- 用碎片时间复习知识库。
- 用问题逼迫自己回忆，而不是只滑动阅读。
- 从卡片回到原文脉络，避免孤立背题。
- 公开知识花园可以成为长期输出和分享入口。

## 3. 产品目标

### 3.1 第一版目标

- 发布一个手机友好的公网知识花园。
- 支持浏览 wiki 页面。
- 支持从 wiki 页面进入复习卡片。
- 支持首页进入今日复习包。
- 支持每日 ingest 后刷新卡片数据。
- 公开来源元数据和原链接，不公开第三方 raw 全文。

### 3.2 工程架构目标

项目采用前后端分离架构，根目录固定包含：

- `backend/`：后端工程目录。
- `frontend/`：前端工程目录。
- `docs/`：开发文档目录。
- `EREADME.md`：项目工程说明文件。

后续开发都在此结构基础上推进。

### 3.3 第一版非目标

- 不做用户登录。
- 不保存个人学习进度。
- 不做真正的间隔重复。
- 不做服务端数据库。
- 不公开第三方 raw 全文快照。
- 不替代本地 Markdown 知识库。

## 4. 信息架构需求

### 4.1 首页

首页应包含：

- 今日复习包入口。
- 最新 ingest 列表。
- 最近更新概念。
- 学习路径入口。
- 概念、摘要、综述、实体导航。

首页在手机端应优先展示“开始复习”入口。

### 4.2 知识花园页

知识花园应公开渲染以下目录中的页面：

- `wiki/concepts/`
- `wiki/entities/`
- `wiki/summaries/`
- `wiki/syntheses/`

每个页面应包含：

- 标题。
- 元数据。
- 正文内容。
- 相关概念或反链。
- 来源链接。
- “复习此页”入口。

### 4.3 卡片复习页

卡片复习页应支持：

- 一次展示一张卡片。
- 默认只展示问题。
- 用户点击后展开答案。
- 展开后显示解释。
- 展示来源 wiki 页面。
- 展示原始来源链接。
- 展示相关概念。
- 支持进入下一张卡片。

### 4.4 今日复习包页

今日复习包应包含：

- 今日新增卡片。
- 从相关旧页面抽取的复习卡片。
- 今日重点概念。
- 今日反常识题或应用题。

如果当天没有合格 ingest，系统可以从已有高价值页面生成旧知识复习包。

### 4.5 来源元数据页

来源元数据页只展示：

- 来源标题。
- 来源类型。
- 信源层级。
- 发布或抓取日期。
- 原始 URL。
- 对应 summary 页面。
- 入库理由。
- 公开引用说明。

来源元数据页不得展示第三方 raw 全文快照。

## 5. 卡片需求

### 5.1 卡片类型

第一版支持五类卡片：

| 类型 | 目的 | 示例问题 |
|---|---|---|
| 定义卡 | 防止概念模糊 | 这个概念是什么意思？ |
| 反常识卡 | 保留 surprise 和信念更新 | 这份资料挑战了哪个直觉？ |
| 对比卡 | 区分相近概念 | A 和 B 的关键差异是什么？ |
| 应用卡 | 连接判断和行动 | 这个知识如何用于产品、工程、学习或判断？ |
| 连接卡 | 激活知识网络 | 它和哪些旧概念发生连接？ |

### 5.2 卡片字段

卡片至少包含：

- `id`
- `type`
- `question`
- `answer`
- `sourcePage`
- `created`

建议包含：

- `explanation`
- `sourceUrl`
- `concepts`
- `difficulty`
- `tags`
- `sourceTier`
- `ingestDate`

### 5.3 卡片生成规则

卡片生成器应优先使用现有 wiki 页面结构：

- 概念页的一句话定义。
- 概念页的核心要义。
- 摘要页的核心论点。
- 综述页的结论和对比。
- 实体页的关系描述。
- `待深入` 和 `相关资料` 区块。

后续 ingest 可以增加 `复习卡片候选` 区块，但第一版不能依赖该区块存在。

### 5.4 卡片质量要求

- 不只生成定义题。
- 每组卡片应尽量包含反常识、对比、应用、连接类问题。
- 卡片答案应短，但解释可以补充上下文。
- 每张卡必须能追溯到至少一个 wiki 页面。
- 有原始 URL 时，卡片应提供原始来源链接。

## 6. 数据与生成流程

### 6.1 数据源

本地 Markdown wiki 是唯一知识源头。

第一版不引入数据库。

### 6.2 生成流程

```text
wiki 页面
  -> 页面解析
  -> 卡片生成
  -> 今日复习包生成
  -> 静态 JSON
  -> 公网前端读取
```

### 6.3 每日 ingest 集成

每日 ingest 完成后，系统应：

1. 识别新增或更新的 wiki 页面。
2. 为相关页面生成卡片。
3. 更新今日复习包。
4. 重建静态 JSON。
5. 保持卡片与来源页面、原始 URL 的链接关系。

### 6.4 开放知识策略

知识范围定位为个人认知操作系统，不强锁 AI/Agent。

每日 ingest 应允许：

- AI/Agent/知识系统/工作台相关内容。
- 产品、工程、组织、商业、认知科学、学习方法、设计、安全等跨域高质量内容。

每日 ingest 应拒绝：

- 低密度新闻。
- 营销稿。
- 泛泛趋势文。
- 与长期学习无关的热点。

跨域资料必须记录入库理由。

## 7. 公网发布与版权边界

### 7.1 可以公开

- wiki 页面。
- 复习卡片。
- 来源元数据。
- 原始来源链接。

### 7.2 不应公开

- 第三方 raw 全文快照。
- 私人笔记。
- 敏感本地路径。
- 标记为私密、未确认或不适合公开的内容。

### 7.3 设计原则

公网产品应成为知识转化与学习入口，而不是第三方文章转载站。

## 8. 非功能需求

### 8.1 移动端优先

第一版应优先保证手机端可读、可点、可复习。

### 8.2 静态优先

第一版应尽量静态化，避免引入服务端复杂度。

### 8.3 可追溯

所有卡片必须保留来源页面。能拿到原始 URL 的，应展示原始 URL。

### 8.4 可扩展

后续应能扩展：

- 用户登录。
- 个人复习进度。
- 间隔重复。
- 主题学习路径。
- NotebookLM 式问答。

但这些不进入第一版。

### 8.5 技术栈约束

后端技术栈：

- Python。
- FastAPI。

前端技术栈：

- Vue 3。
- TypeScript。
- Tailwind CSS。
- Lucide icon。
- shadcn/ui 标准 UI 组件库和样式。
- inspira-ui 动效框架，按需用于页面动效。

### 8.6 UX 原则

产品 UX 必须遵循尼尔森十大交互原则：

- 系统状态可见。
- 系统与现实世界匹配。
- 用户控制与自由。
- 一致性和标准。
- 防错。
- 识别优于回忆。
- 灵活高效。
- 美学与极简设计。
- 帮助用户识别、诊断并恢复错误。
- 帮助与文档。

## 9. 验收标准

第一版完成时，应满足：

- 公网手机端能打开网站。
- 首页能进入今日复习包。
- 能浏览公开 wiki 页面。
- 任意支持卡片的 wiki 页面能进入“复习此页”。
- 卡片页能展示问题、展开答案、查看解释、进入下一张。
- 卡片能回链到来源 wiki 页面。
- 来源元数据页只展示元数据和原链接，不展示第三方 raw 全文。
- 每日 ingest 后能刷新卡片数据和今日复习包。
- 网站不依赖登录和数据库即可运行。

## 10. 后续版本方向

### 10.1 V2 候选

- 本地或账号级学习进度。
- 真正的间隔重复。
- 学习路径页。
- 主题复习包。
- 卡片收藏。
- 卡片纠错反馈。

### 10.2 V3 候选

- NotebookLM 式主题问答。
- 根据错题自动生成补充卡。
- 公开知识图谱视图。
- 自动生成文章、决策备忘录或学习报告。
