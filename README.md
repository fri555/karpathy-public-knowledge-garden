# 📖 README — Karpathy 式知识库使用说明

> 这是按 [Andrej Karpathy 的 LLM Wiki 理念](https://gist.github.com/karpathy) 搭建的个人知识库。
> 给您（人类用户）看的，不是给 Agent 看的。Agent 请读 [[AGENT]]。

---

## 🎯 一句话原理

**不要把 LLM 当搜索引擎，要把它当一个不知疲倦的知识工程师**——你扔资料、提问题，它负责把知识**编译**成一个持续生长的 Wiki，而不是问完即忘。

---

## 🧩 三个核心动作

跟 Hana 说这三句话之一就能驱动整个系统：

### 1️⃣ Ingest — 录入资料

> **"Hana，帮我把 xxx 录入知识库"**
> （xxx 可以是：一份 PDF、一篇文章链接、一段对话、一张图、甚至一句你今天的想法）

Hana 会：把原始资料存进 `raw/` → 写一份摘要 → 抽出涉及的人、公司、概念 → 更新或新建对应的 wiki 页 → 把所有相关页用 `[[双向链]]` 串起来 → 更新总索引 → 写日志。

一份资料通常会牵动 5–15 个 wiki 页面更新。

### 2️⃣ Query — 提问

> **"Hana，关于 xxx，我的知识库里都有什么？"**
> **"Hana，对比一下 xxx 和 yyy"**
> **"Hana，根据我的知识库，xxx 怎么做最合理？"**

Hana 会：先读 `wiki/index.md` 定位 → 综合相关页回答 → 回答里带 `[[页面链接]]` 让你能跳过去 → **把新产生的洞察沉淀回 wiki**（这是关键，问完即忘 = 系统失败）。

### 3️⃣ Maintain — 体检

> **"Hana，给知识库做个审计"**

Hana 会输出一份审计报告，检查：矛盾、过时数据、孤儿页、缺失反链、隐藏概念、空壳页，并建议你下一步该研究什么。

建议每录入 10 份左右资料做一次。

---

## 🗂️ 目录是什么样的

```
【K】Karpathy知识库/
├── AGENT.md          ← Agent 协议（精髓在这）
├── README.md         ← 你正在读的
├── raw/              ← 原始资料（Agent 只读，不会被改）
├── wiki/             ← 知识层
│   ├── index.md      ← 总索引（要找东西从这里开始）
│   ├── log.md        ← 操作日志（每次 Agent 干了什么都有记录）
│   ├── entities/     ← 人 / 公司 / 产品 / 组织
│   ├── concepts/     ← 术语 / 方法 / 理论
│   ├── summaries/    ← 单份资料的精读摘要
│   └── syntheses/    ← 跨资料的综述 / 对比 / 审计报告
└── .templates/       ← 模板（可绑到 Obsidian 模板插件，或忽略）
```

## ✅ 现在可以做的第一件事

最自然的起步方式 ——

把您引发这个项目的两篇知乎文章作为第一批资料录入。直接跟 Hana 说：

> **"Hana，把那两篇知乎文章录入知识库"**

Hana 会按 [[AGENT]] 协议跑一遍完整的 ingest 流程，您就能亲眼看到一份资料是怎么被"编译"成 wiki 页的。

---

## 📝 给您的几个小提示

1. **不用学 Obsidian 复杂操作**：所有 `[[双向链]]` 都是 Agent 维护，您只管读和点
2. **想看知识网络全貌**：打开 Obsidian 图谱视图，定位到 `【K】Karpathy知识库/` 这个文件夹，就能看到所有页面怎么连
3. **`.templates/` 里有三个模板**：如果您装了 Obsidian Templates 插件，可以把模板目录指过去，手动创建页时也能用
4. **想看 Agent 干过啥**：直接看 [[wiki/log]]
5. **想改规则**：跟 Hana 商量，确认后会改 [[AGENT]] 并在 log 留痕

---

## ⚠️ 边界

- Hana 只在 `【K】Karpathy知识库/` 这个目录内读写，**绝不动您其他工作笔记**（【A】~【M】、AI相关内容、copilot 等都不会被碰）
- `raw/` 是不可变层，Hana 不会改原文，只在 wiki 层批注
- 任何不确定的事，Hana 会停下来问您，不会瞎写
